What is SCIM?
System for Cross-domain Identity Management

SCIM (System for Cross-domain Identity Management) is an open standard that automates user provisioning between your identity provider and SaaS applications. Instead of manually creating/deleting users in each app, SCIM does it automatically.

❌ Without SCIM
  • Manual user creation in every app
  • Delayed offboarding (security risk)
  • Inconsistent user attributes
  • IT tickets for every change
  • Orphaned accounts accumulate
✅ With SCIM
  • Users auto-provisioned on hire
  • Instant deprovisioning on exit
  • Consistent profile data everywhere
  • Zero manual IT effort
  • Audit trail for compliance
SCIM 2.0 is defined in RFC 7642, RFC 7643, and RFC 7644. Your TerzoCloud SCIM server implements the full SCIM 2.0 protocol at /scim/v2/ endpoints.

What happens when a user is added?

1
User created in TerzoCloud portal
HR adds a new employee in the User Management section. The user record is saved to MongoDB.
2
SCIM event triggered
The portal calls POST /scim/v2/Users with the user's profile data (name, email, department).
3
Connector routing
The SCIM server checks which App Connectors are enabled and routes the provisioning request to each one.
4
User created in each app
Each connector calls the target app's API to create the user account (e.g. GitHub API, Slack API, Zoom API).
5
Confirmation logged
The SCIM server logs the result. Success or failure is visible in the Admin → Connectors panel.

System Requirements

  • TerzoCloud Portal running at http://localhost:3000 with MongoDB connected
  • Admin account with Super Admin role (SCIM endpoints require onlySuperAdmin middleware)
  • Node.js 18+ and the AppConnector Mongoose model deployed

SCIM Server Configuration
Setting up your TerzoCloud SCIM server

The TerzoCloud SCIM server is built into server.js and runs on the same Node.js/Express process as the main API. No additional setup is needed — just make sure the server is running and you are authenticated as Super Admin.

Environment Variables

Ensure your .env file contains the following:

# MongoDB connection MONGODB_URI=mongodb://localhost:27017/terzocloud # JWT secret (keep this secret!) JWT_SECRET=your-super-secret-key-here # Port (default 3000) PORT=3000 # Optional: encryption key for stored API tokens TOKEN_ENCRYPTION_KEY=32-char-random-string-here

Starting the Server

# Navigate to your IT-SYSTEM directory cd IT-SYSTEM # Install dependencies (first time) npm install # Start the server node server.js # Or with nodemon for development npx nodemon server.js
Once running, the SCIM server is accessible at http://localhost:3000/scim/v2/. You can test it with a GET request to /scim/v2/ServiceProviderConfig.

Verifying the Server

# Test SCIM Service Provider Config endpoint curl http://localhost:3000/scim/v2/ServiceProviderConfig # Should return JSON with supported SCIM operations # Expected: {"schemas":["urn:ietf:params:scim:schemas:core:2.0:ServiceProviderConfig"],...}

Authentication
How the SCIM server authenticates requests

All connector management endpoints require a valid JWT token with Super Admin privileges. Get your token by logging in through the portal.

# Login to get a JWT token curl -X POST http://localhost:3000/api/login \ -H "Content-Type: application/json" \ -d '{"email":"admin@terzocloud.com","password":"yourpassword"}' # Use the returned token in subsequent requests curl http://localhost:3000/api/admin/connectors \ -H "Authorization: Bearer <your_jwt_token>"
API tokens for each app connector are stored encrypted in MongoDB. They are never returned in API responses — only the last 4 characters are shown as a hint.

Zoom Connector
Sync licensed user count from Zoom account
✓ SCIM Ready Plan: Workspace Business Plus API: REST v2

What this connector does

The Zoom connector syncs the number of active licensed users and plan details from your Zoom account into the Software Inventory. When a user is offboarded, the connector can also remove their Zoom license automatically.

Step-by-step Setup

1
Go to Zoom Marketplace
Navigate to marketplace.zoom.us → Develop → Build App → Choose Server-to-Server OAuth.
2
Create a Server-to-Server OAuth App
Name it "TerzoCloud SCIM". Copy the Account ID, Client ID, and Client Secret.
3
Add Required Scopes
Under Scopes, add: user:read:admin and user:write:admin. These are required to read and manage users.
4
Activate the app
Click Activate your app in the Marketplace. Generate an Account-Level OAuth token using your Client ID and Client Secret.
5
Configure in TerzoCloud
Go to Admin → App Connectors → Add Connector → Zoom. Paste your OAuth token and Account ID, then click Save.

Connector Fields

Account-Level OAuth Token
Bearer token generated from your Server-to-Server OAuth app. Used for all API calls.

Account ID
Your Zoom Account ID from the Server-to-Server OAuth app settings.
ABC123XYZ

GitHub Connector
Sync active member count from your GitHub organisation
✓ SCIM Ready Plan: Enterprise Cloud API: GitHub REST v3

What this connector does

The GitHub connector reads the number of active members in your GitHub organisation and syncs the count to the Software Inventory. With SCIM, it can also auto-provision new engineers and remove access when someone leaves.

Step-by-step Setup

1
Go to GitHub Settings
Navigate to github.com/settings/tokens → Generate new token (classic).
2
Set Token Scopes
Select scope: read:org (to read org member count). For full SCIM provisioning, also add admin:org and scim:enterprise.
3
Copy the token
GitHub shows the token only once. Copy it immediately — you cannot view it again.
4
Configure in TerzoCloud
Go to Admin → App Connectors → Add Connector → GitHub. Enter your Organisation slug (e.g. terzocloud) and paste the PAT.

Connector Fields

Personal Access Token (PAT)
Classic or fine-grained PAT from GitHub Settings → Developer Settings → Personal access tokens.
ghp_xxxxxxxxxxxxxxxxxxxx
Organisation Slug
Your GitHub organisation name as it appears in the URL (case-sensitive).
terzocloud
Full SCIM provisioning requires GitHub Enterprise Cloud. The sync feature (reading member counts) works with any GitHub plan.

Google Workspace Connector
Sync users via Service Account and Admin SDK
✓ SCIM Ready Plan: Business Starter+ API: Admin SDK Directory

What this connector does

Uses Google's Admin SDK Directory API to read the total number of active users in your Google Workspace domain, and optionally provision/suspend users automatically.

Step-by-step Setup

1
Create a Google Cloud Project
Go to console.cloud.google.com → Create a new project (e.g. "TerzoCloud SCIM").
2
Enable Admin SDK API
In your project: APIs & Services → Enable APIs → Search for "Admin SDK API" → Enable.
3
Create a Service Account
IAM & Admin → Service Accounts → Create service account. Download the JSON key file.
4
Enable Domain-Wide Delegation
In the service account settings, enable Domain-wide delegation. Then in your Google Workspace Admin Console: Security → API Controls → Domain-wide delegation → Add your service account's Client ID with scope https://www.googleapis.com/auth/admin.directory.user.readonly.
5
Configure in TerzoCloud
Go to Admin → App Connectors → Google Workspace. Paste the full service account JSON key and enter your Admin email (e.g. admin@yourcompany.com).

Connector Fields

Service Account Key JSON
The full JSON key file downloaded from Google Cloud Console. Contains the private key for API authentication.
{"type":"service_account","project_id":"terzocloud-scim","private_key":"-----BEGIN RSA PRIVATE KEY-----…"}
Admin Email (for impersonation)
A Google Workspace Super Admin email. The service account will impersonate this user to call the Directory API.
admin@terzocloud.com

Configure in TerzoCloud

Microsoft 365 Connector
Sync users via Azure AD / Microsoft Entra ID
✓ SCIM Ready Plan: Any M365 paid plan API: Microsoft Graph

What this connector does

Uses the Microsoft Graph API to read the number of licensed Microsoft 365 users in your Azure Active Directory / Entra ID tenant and syncs the count to Software Inventory.

Step-by-step Setup

1
Register an App in Azure Portal
Go to portal.azure.com → Azure Active Directory → App registrations → New registration. Name it "TerzoCloud SCIM".
2
Copy Tenant ID and Client ID

Create a Client Secret
Under Certificates & secrets → Client secrets → New client secret. Copy the Value (not the Secret ID). It's only shown once.

Grant API Permissions
API permissions → Add permission → Microsoft Graph → Application permissions → Add: User.Read.All. Then click Grant admin consent.
5
Configure in TerzoCloud
Go to Admin → App Connectors → Microsoft 365. Enter your Tenant ID and paste the Client Secret Value.

AWS IAM Identity Center Connector
Sync users via SCIM 2.0 from AWS Identity Center
✓ SCIM Ready Plan: Any AWS account API: SCIM 2.0 Native

What this connector does

AWS IAM Identity Center has native SCIM 2.0 support. This connector reads the number of provisioned users and can push user creation/deletion events directly to AWS.

Step-by-step Setup

1
Open IAM Identity Center
In the AWS Management Console, search for IAM Identity Center (formerly AWS SSO). Enable it if not already active.
2
Enable Automatic Provisioning
Settings → Identity source → Actions → Enable automatic provisioning. This reveals the SCIM endpoint URL and a bearer token.
3
Copy SCIM Endpoint URL and Token
Copy both the SCIM endpoint URL (e.g. https://scim.us-east-1.amazonaws.com/xxxx/scim/v2) and the Access token. The token is only shown once.
4
Configure in TerzoCloud
Go to Admin → App Connectors → AWS IAM Identity Center. Paste the SCIM Endpoint URL as the org field and the Access token as the token field.

Connector Fields

SCIM Bearer Token
The access token generated by AWS when enabling automatic provisioning. Only shown once — store it securely.
eyJr… (long token)
SCIM Endpoint URL
The full SCIM endpoint URL from your IAM Identity Center settings. This is where TerzoCloud will send user provisioning requests.
https://scim.us-east-1.amazonaws.com/xxxxxxxx/scim/v2
AWS IAM Identity Center provides native SCIM 2.0 — no extra plan required. It works on any AWS account (including Free Tier).

Frequently Asked Questions
Common questions about SCIM setup
❓ How often does the connector sync?

The SCIM Audit auto-refreshes every 60 seconds. Manual syncs happen when you click "Sync Now". Full automated provisioning (user create/delete) is triggered by user events in the portal.

❓ Are API tokens stored securely?

Yes — tokens are stored encrypted in MongoDB. Only the last 4 characters are displayed in the UI. Setting TOKEN_ENCRYPTION_KEY in your .env enables AES encryption at rest.

❓ Can I have multiple connectors active at the same time?

Yes — all 6 connectors can be enabled simultaneously. Each connector operates independently and only affects its own target app.

❓ What happens if a connector sync fails?

The connector status dot turns red and the last error message is stored. The Software Inventory count is not updated. Fix the issue and click "Test Connection" then "Sync Now".

❓ Do I need all 6 connectors?

No — configure only the ones you need. Each connector is fully optional and independent. You can add or remove connectors at any time from Admin → App Connectors.
